# 게임팩 전면 재점검 (냉정 감사) — 2026-07-18

> 목적: 응원이 아니라 **냉정한 진단**. 강점은 짧게, 문제는 깊게.
> 대상: `system/`(코어 64모듈·5906줄) + 컨셉 서사 + 앱 16화면.

## 0. 한 줄 총평

**엔진 설계는 A급, 운영 설계는 어중간하다.** 잘 만든 전투/성장 엔진 위에
**시스템을 계속 쌓기만 해서**(40+ 모듈), 정작 출시 설정에선 핵심 두 축을 꺼뒀고,
중복된 "파워 그릇"들이 겹쳐 있으며, 서사는 껍데기다. **깊이보다 개수로 커졌다.**

---

## 1. 강점 (짧게, 사실만)

- **3레이어 아키텍처**(core/genre/concept, 단방향 의존) — IP=시스템 개념이 실제로 동작. 판타지↔SF 스왑 구현됨. 재사용 설계는 모범적.
- **`resolve()` 판정 엔진** — TTK(처치시간) 비교 모델. 하나의 순수 함수가 RPG(win)·방치(duration)·PvP·DPS미터를 전부 지탱. 결정론적 → 서버 재검증(안티치트) 가능.
- **시뮬 기반 밸런스** — 7일 오토플레이 시뮬로 "지수적 벽"을 찾아 환생 배수로 해소. 근거 있는 튜닝(`balance-report.html`).
- **저장 견고성** — SAVE_VERSION·백업·로스터 마이그레이션.

→ **엔진과 인프라는 신뢰할 만하다.** 문제는 그 위다.

---

## 2. 냉정한 문제점 (핵심)

### P1. 출시 설정과 빌트 시스템의 괴리 — 만든 걸 꺼뒀다
`features.mjs` 기본값: **`elements`(속성 상성)·`rarity`(등급 N~UR) 둘 다 OFF**
(`features.test.mjs`가 명시 단언). 완성·테스트된 두 **헤드라인 가챠 시스템**이
주력(판타지) 출시 모드에서 **비활성**이다.
- `affinity()`는 항상 1 반환 → 속성 상성(FIRE>WOOD…) 무의미.
- 등급 배수 1.0 고정 → 가챠 천장(90회 SSR·UR 0.5%)이 도는데 **등급이 파워를 안 준다**.
  = "고등급 뽑아 강해지기"라는 가챠 RPG의 핵심 동기가 힘을 잃음.
- 판타지 로스터엔 아예 `rarity/element` 필드조차 없음(SF만 있음).

**진단**: 지금은 "단순 코어"와 "풀 가챠" 사이 **어중간한 상태**. 만든 복잡도의 상당분이 잠들어 있다.

### P2. 중복된 "파워 그릇" — feature-stacking
성장 요소가 **평행 축으로 과다 분화**돼 있다. 특히 계정 파워:
| 축 | 시스템 | 귀결 |
|---|---|---|
| 계정 파워 | 환생 · **relics** · **emblems** · **guardians** | 전부 `accountMods.powerMult`로 합산 |
| 유닛 파워 | gear · runes · sigweapon · seed · starGrade · enhance/awaken · intimacy | 전부 `modifiers`로 합산 |

`emblems.mjs` 주석이 자백한다: *"유물과 형제지만 축이 다르다."*
→ 플레이어 관점에선 **"모으고 장착해 숫자 올리는" 같은 행위의 반복**. 축 간 차별성이 약해
인지 부하만 크다. 4개 계정-파워 축, 7개 유닛-파워 축은 소규모 게임엔 과하다.

### P3. 전투 복잡도 vs 체감
`resolve()`가 다루는 스탯 상호작용 **13종+**(감쇠·관통·절대공격·흡혈·피해감소·절대방어·
회피·명중·팀공/팀치명/팀방·시너지·진형·상성), 각기 상한(0.5/0.6/0.9).
- 밸런싱 표면이 넓어 **취약**(하나 바꾸면 연쇄).
- 방치형은 대부분 자동 전투 → 이 깊이를 **플레이어가 체감·전략화하기 어렵다**. 불투명한 파워.

### P4. 시나리오는 껍데기
- 캠페인 12챕터 = **보스 앞 1~2문장 flavor**. 플롯 내 등장인물·대사씬·분기 없음. 빌런은 무명 원형.
- **판타지 캐릭터 대사 0개**(SF는 전원 보유). 로스터에서 말풍선 안 뜸.
- 씨앗 서사 = 성장 라벨(플롯 아님).
→ "수집형 RPG"를 표방하나 **캐릭터에 서사·개성이 없다**(주력 판타지 기준).

### P5. 컨셉 간 불균형 — 주력이 더 빈약
SF가 판타지보다 캐릭터 데이터(대사·성격·별칭·rarity·element)가 **풍부**하다.
정작 기본 출시(판타지)가 더 앙상하다. 쇼케이스와 실제 제품이 뒤바뀐 셈.

### P6. 콘텐츠 루프 과다
방치 코어 위에 캠페인·무한탑·아레나·길드·원정(로그라이트)·시즌·이벤트·일일.
각각 밸런스·보상곡선·UI를 요구한다. **소규모 유지보수로 다 살아있게 할 수 있는가**는 의문.
(원정은 이번에 UI까지 붙였지만, 나머지 다수는 얕게 존재.)

## 4. 냉정한 결론 & 우선순위 제언

**핵심 모순**: "시스템이 IP"라는 좋은 사상을 **시스템 개수로 오해**해, 다듬음보다 적재가 앞섰다.
지금 필요한 건 **더 만들기가 아니라 정리·확정·심화**다.

### 먼저 결정할 것 (방향)
게임이 **어느 쪽 제품인지 확정**해야 한다. 지금은 어중간하다.
- (A) **단순 코어 확정** — elements/rarity off 유지 + 중복 파워축(relics/emblems/guardians) 통합·감축. 얕은 콘텐츠 루프 정리. → 가볍고 명확한 방치형.
- (B) **풀 가챠 확정** — elements/rarity ON + 판타지에 rarity/element/대사 데이터 채우고 전면 밸런싱. → 정통 수집형 RPG. 작업량 큼.

### 우선순위 (방향 무관 공통)
1. 🔴 **정체성 결정(A/B)** — 나머지가 여기 종속.
2. 🔴 **파워 축 통합**(P2) — relics·emblems·guardians를 1~2축으로. 즉시 명료도↑.
3. 🟡 **서사 최소 채우기**(P4/P5) — 판타지 캐릭터 데이터 SF 수준 동등화(저비용).
4. 🟡 **콘텐츠 루프 취사선택**(P6) — 살릴 것만 남기고 나머지 플래그 off로 숨김.
5. 🟢 **전투 스탯 다이어트**(P3) — 체감 없는 상호작용 정리(B 노선이면 유지·튜닝).

> 요약: **잘 만든 엔진, 과적된 상부구조, 빈 서사.** 빼고 확정하는 게 지금의 리뉴얼이다.

# 인앱 GUI/UI 골격 명세서 — 세븐나이츠 최종본 기준

> 기준 레퍼런스: **세븐나이츠 키우기 최종 목업**
> (https://gimaxax77-sys.github.io/elrogue-play/seven-real.html · 실제 스크린샷 정정본).
> 상위 설계: [`UI_ARCHITECTURE.md`](../../axdata_engine_엘로그2_cocos/UI_ARCHITECTURE.md)(3계층 조립식).
> 대상: 인앱(RN/Expo, axdata_03) 적용. 좌표/치수는 **세로 모바일(폭 360~430)** 기준 권장값.
> 원칙: **기본 골격 = 항상 로드**, 나머지 = **슬롯 배열 + feature 플래그로 조립**([[UI_ARCHITECTURE]]).

│  │                                     │t │
│R │                                     │  │
│a │─────────────────────────────────────│C │
│i │        G4c ActionDock (하단 액션)      │a │
│l │       수령·소탕·자동토글 (오버레이)       │rd│

## 1. G1 · AppShell (최상위 프레임) — **항상 로드**
| 항목 | 명세 |
|---|---|
| 역할 | 세이프에어리어 + 배경(그라데이션) + wide(태블릿/PC) 시 폰폭 가운데 정렬 |
| 구성 | SafeAreaView → 배경 LinearGradient(absoluteFill) → `frame`(wide면 maxWidth 480 center) |
| 슬롯 | 자식 존 G2/G3/G4/G5를 순서대로 배치 |
| 데이터 | 없음(레이아웃 전용) |
| 현행(axdata_03) | App.js가 이미 구현(SafeAreaView+bgGrad+frameWide). **재사용** |

## 2. G2 · TopBar (상단 재화·프로필·진행) — **항상 로드, 슬롯형**
세븐나이츠: 프로필(닉·골드 4.2M·다이아 980)·설정·모험 92/120·2배속·선물타이머.

| 슬롯 | 내용 | 고정/플래그 | 데이터 | 비고 |
|---|---|---|---|---|
| `profile` | 아바타+닉네임+레벨 | 플래그 `profile` | state.player | 없으면 생략 가능 |
| `currency` | 재화 2~4종(골드·정수·소환석·다이아) | 고정(최소 2) | wallet | 현행 ResourceBar 재사용 |
| `progress` | 진행 지표(스테이지 x/y · 배속 · 타이머) | 플래그 `stageHeader` | state.stage | 세븐나이츠식 상단 진행 |
| `system` | 설정·우편·알림 아이콘 | 고정(설정)·플래그(우편=mail) | mailUnread | 현행 📬 재사용 |

- 높이 56~64. 재화 셀은 아이콘22+값. "받을 것" 펄스/뱃지.
- 치수: 좌 profile(flex) · 중 currency(내용폭) · 우 system(아이콘 열).
- **적응형**: 재화 종류 수는 concept가 정의(2~4). progress 슬롯 off면 높이 유지·해당 칸만 비움.

## 3. G3 · StageView (중앙 컨텍스트 뷰) — **항상 로드, 렌더러 주입**
세븐나이츠: 중앙이 편성10슬롯+AUTO / 소환(영웅·펫 탭·270/300·1·10·30연·배너). = **탭/컨텍스트별로 중앙이 바뀜.**

| 모드 | 중앙 내용 | 진입 |
|---|---|---|
| `battle`(기본) | 자동 전투 렌더(파티 vs 적, HP바·데미지·연출) | 전투 탭 |
| `roster` | 편성 슬롯 + 로스터 그리드 | 영웅 탭 |
| `summon` | 소환 배너+당기기(1/10/30)+진행바 | 소환 탭 |
| (기타) | 콘텐츠/상점 등 화면이 중앙을 채움 | 각 탭 |

- **핵심**: StageView는 "무엇을 그릴지"를 **주입**받는다(렌더러/화면 컴포넌트). 트랙별 렌더러 교체 지점(RN=BattleView, Cocos=씬).
- 배경: 스테이지 배경 이미지 로테이션(층/구역별).
- 치수: G2와 G5 사이 **남는 세로 전부**(flex:1). 좌우 레일은 이 위에 **오버레이**(absolute), 중앙 폭을 잠식하지 않음.

## 4. G4a · LeftRail (좌측 기능 레일) — **플러그인 슬롯 (오버레이)**
세븐나이츠 정정 포인트: 좌측은 "현황판"이 아니라 **세로 기능버튼 열**(퀘스트·배속·보상타이머·이벤트).

| 명세 | 값 |
|---|---|
| 배치 | StageView 좌측에 absolute, 세로 스택, 폭 44~56 |
| 구성 | 아이콘 버튼(뱃지 지원) 배열 = `LEFT_RAIL[]` 레지스트리 |
| 항목(예) | 퀘스트·일일·이벤트·배속·우편·공지 (각 `{id,icon,badge,onPress,feat}`) |
| 고정/플래그 | 전부 플래그. 켜진 것만 위→아래 정렬 |
| 데이터 | 각 버튼이 자체 뱃지 카운트 계산 |
| 적응형 | 항목 0개면 레일 자체 미렌더(중앙 폭 확보) |

## 5. G4b · RightCards (우측 콘텐츠/던전 카드) — **플러그인 슬롯 (오버레이)**
세븐나이츠: 우측 던전 카드(골드·영웅경험치·기사토큰·아티팩트) + 쿨다운/진행(2/3).

| 명세 | 값 |
|---|---|
| 배치 | StageView 우측 absolute, 세로 카드 스택, 폭 64~84 |
| 구성 | 미니 카드 배열 = `RIGHT_CARDS[]`(`{id,icon,label,progress,cooldown,onPress,feat}`) |
| 항목(예) | 장비던전·룬던전·요일던전·펫던전·시즌·이벤트던전 |
| 고정/플래그 | 전부 플래그(콘텐츠 모듈과 1:1: dungeon*/season/events…) |
| 데이터 | 입장 가능/쿨다운/진행도 |
| 적응형 | 카드 0개면 레일 미렌더 |

## 6. G4c · ActionDock (하단 액션) — **항상 로드, 액션 레지스트리**
방치형 필수(UI_ARCH G4): 방치보상 수령·소탕·자동토글. 세븐나이츠 AUTO 토글 포함.

| 명세 | 값 |
|---|---|
| 배치 | StageView 하단(중앙 하단 오버레이) 또는 G5 위 얇은 바 |
| 구성 | 액션 버튼 배열 = `DOCK_ACTIONS[]`(`{id,icon,label,onPress,feat,primary}`) |
| 기본 항목 | 방치보상 수령(항상) · AUTO 토글(항상) · 소탕(플래그 `sweep`) · 배속(플래그 `speed`) |
| 강조 | `primary` 액션(수령)은 크게/골드. 획득 시 숫자 팝업 |
| 적응형 | 최소 "수령+AUTO"는 유지 |

## 7. G5 · BottomNav (하단 2단 내비) — **항상 로드, 탭 레지스트리**
세븐나이츠: 상단 서브탭(맥락) + 고정 아이콘바(모험·영웅·소환·장비·던전).

### 7-1. G5b TabBar(메인 탭) — 고정
| 명세 | 값 |
|---|---|
| 소스 | `ALL_TABS[]` 레지스트리(현행 App.js에 이미 존재) → `isOn(feat)` 필터 |
| 현행 탭 | 전투·영웅·소환·원정·콘텐츠·상점 (feat: gacha/expedition/shop 등) |
| 구성 | 아이콘+라벨+활성 인디케이터+뱃지(●) |
| 개수 | 5~6 권장(초과 시 더보기). 세븐나이츠=5(모험·영웅·소환·장비·던전) |
| 적응형 | `feat` off 탭 자동 제외(현행 구현됨) |

### 7-2. G5a SubTabBar(서브탭) — 조건부
| 명세 | 값 |
|---|---|
| 소스 | 화면별 서브탭 레지스트리(예: `ROSTER_SUBTABS`, `CONTENT_SUBTABS`) → `isOn` 필터 |
| 표시 | 해당 탭에 서브탭이 2개 이상일 때만 노출(1개면 숨김) |
| 현행 | ContentScreen SUBTABS·RosterScreen ROSTER_TABS/DETAIL_TABS 존재(부분) — **레지스트리로 통일 필요** |

## 8. 3계층 조립 레지스트리 (구현 규약)
`UI_ARCHITECTURE.md` 계층을 인앱 파일로 구체화.

| 계층 | 파일(제안) | 형태 | 추가/제거 |
|---|---|---|---|
| 1 탭 | `app/registry/tabs.js` (=현 ALL_TABS 이관) | `{key,label,icon,Screen,feat}` | 배열 push/삭제·feat |
| 2 위젯 슬롯 | `app/registry/leftRail.js`·`rightCards.js`·`dockActions.js`·`*Subtabs.js` | `{id,icon,…,feat}` | 배열 편집 |
| 3 플래그 | `system/core/features.mjs`(존재) | `FEATURES{key:bool}` + `isOn` | 값 토글 |

- 각 존 컴포넌트는 **해당 레지스트리 배열을 `.filter(isOn).map(render)`** 로만 그린다(하드코딩 금지).
- 트랙별 게임성 = **플래그셋 + 레지스트리 내용**만 다르게.

## 9. 적응형 규칙 (모듈 off 시 재구성) — **GUI 토킷 완성 조건**
1. **존 단위 collapse**: LeftRail/RightCards/SubTab은 유효 항목 0이면 **존 자체 미렌더** → 중앙이 그 폭을 회수.
2. **탭 축소**: `ALL_TABS` 필터로 빈 탭 제거(현행). 남은 탭이 4개 미만이면 라벨 크게.
3. **최소 보장**: G1·G2(재화 최소2)·G3·G4c(수령+AUTO)·G5b(탭≥2)는 항상 유지 — 이하로는 게임 성립 안 함(=하드코어 뼈대).
4. **빈자리 금지**: off된 슬롯은 숨김이지 빈 박스가 아니다(레이아웃 재계산).

## 11. 우선 구현 순서(제안)
1. `registry/` 폴더 + `tabs.js`(ALL_TABS 이관, 무동작 리팩터).
2. `dockActions.js` + ActionDock 존(방치수령·AUTO) — 방치형 핵심.
3. `leftRail.js`·`rightCards.js` 슬롯 + 존 렌더(빈 항목 collapse).
4. TopBar 슬롯화(profile/progress).
5. 서브탭 레지스트리 통일(Content/Roster).
6. 적응형 규칙(§9) 검증: 플래그 조합별 레이아웃 스냅샷.
